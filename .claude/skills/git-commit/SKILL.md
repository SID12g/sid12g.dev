# Git Commit Creator

## 목표

변경된 파일을 분석하여 **의미 있는 단위로 그룹핑**하고, 각 그룹에 맞는 커밋 메시지를 제안한 뒤 사용자 승인 후에만 커밋을 실행합니다.

---

## 실행 순서

### 1단계: 저장소 상태 파악

```bash
git status
git diff --stat HEAD
```

- 변경된 파일 목록과 각 파일의 변경 규모(추가/삭제 라인 수)를 확인합니다.
- Untracked 파일과 Staged/Unstaged 파일을 구분합니다.
- 변경 사항이 없으면 사용자에게 알리고 종료합니다.

### 2단계: 변경 내용 분석

파일별로 실제 변경 내용을 확인합니다:

```bash
git diff HEAD -- <파일경로>          # 이미 추적 중인 파일
git diff -- <파일경로>               # Staged 변경
git diff --cached -- <파일경로>      # Staged 변경 (동의어)
```

Untracked 파일은 내용을 직접 읽어 어떤 파일인지 파악합니다.

### 3단계: 그룹핑

변경 파일들을 **하나의 목적을 가진 커밋 단위**로 묶습니다.

**그룹핑 기준 (우선순위 순)**:

1. **기능 단위**: 같은 기능에 속하는 파일들 (예: 컴포넌트 + 스타일 + 테스트)
2. **변경 유형**: 리팩토링, 버그 수정, 설정 변경 등 성격이 같은 파일들
3. **영향 범위**: 서로 독립적이고 다른 목적을 가진 파일은 별도 커밋으로 분리

**그룹당 파일 수 가이드**: 1~5개가 이상적. 관련 없는 파일을 억지로 묶지 않습니다.

### 4단계: 커밋 계획 보고

아래 형식으로 커밋 계획을 보고합니다:

```
📊 변경 사항 분석
• 변경된 파일: N개 (수정 N / 추가 N / 삭제 N)

🔄 커밋 계획 (총 N개)

커밋 1/N
  타입: feat
  Subject: feat: 로그인 폼 유효성 검사 추가
  Body: - 이메일 형식 검증 로직 구현
        - 빈 필드 제출 방지 처리
  파일: src/components/LoginForm.tsx
        src/utils/validators.ts

커밋 2/N
  ...

🚫 제외된 파일 (해당 시)
  • .DS_Store — OS 자동 생성 파일, 커밋 불필요
```

### 5단계: 사용자 승인 대기

커밋 계획을 보고한 후 **반드시 사용자의 명시적 승인을 기다립니다**.

승인 전에는 어떤 git 명령도 실행하지 않습니다.

### 6단계: 커밋 실행 (승인 후)

승인을 받은 경우, 그룹 순서대로 커밋을 실행합니다:

```bash
git add <파일1> <파일2> ...
git commit -m "<Subject>" -m "<Body>"
```

모든 커밋 완료 후 push 여부를 확인합니다:

```bash
git push origin <현재 브랜치>   # 사용자가 원하는 경우
```

## 커밋 메시지 형식

[Conventional Commits](https://www.conventionalcommits.org/) 형식을 따릅니다.

### Subject (첫 번째 줄)

```
<타입>(<스코프>): <요약>
```

- **50자 이내**
- 현재형 동사로 시작 (추가, 수정, 제거 / add, fix, remove)
- 마침표 없음
- 스코프는 생략 가능

### Body (두 번째 `-m` 인자)

- **각 줄 72자 이내**
- 무엇을, 왜 변경했는지 설명
- 불릿(`-`)으로 항목 구분 가능
- 생략 가능 (단순한 변경의 경우)

### 타입 목록

| 타입       | 사용 시점                       |
| ---------- | ------------------------------- |
| `feat`     | 새로운 기능 추가                |
| `fix`      | 버그 수정                       |
| `refactor` | 기능 변경 없는 코드 개선        |
| `style`    | 포맷, 세미콜론 등 스타일만 변경 |
| `docs`     | 문서 추가/수정                  |
| `test`     | 테스트 추가/수정                |
| `chore`    | 빌드, 패키지, 설정 변경         |
| `perf`     | 성능 개선                       |
| `ci`       | CI/CD 설정 변경                 |

### 커밋 메시지 예시

```
feat(auth): 소셜 로그인 버튼 컴포넌트 추가

- Google, GitHub OAuth 버튼 UI 구현
- 로그인 실패 시 에러 메시지 표시
```

```
fix: 다크모드에서 텍스트 색상 깨지는 문제 수정
```

## 제약 사항

- **사용자 승인 전 커밋 금지**: 계획을 보고하고 반드시 승인을 받은 후 실행합니다.
- **억지 그룹핑 금지**: 관련 없는 파일은 묶지 말고 별도 커밋으로 분리합니다.
- **Co-Authored-By 금지**: 커밋 메시지에 추가하지 않습니다.
- **불필요한 파일 제외**: `.DS_Store`, `*.log`, `node_modules/` 등은 커밋에서 제외하고 사용자에게 알립니다. `.gitignore` 추가를 제안할 수 있습니다.
