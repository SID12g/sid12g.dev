### 1. 개요

본 프로젝트는 같은 고등학교 학생들이 모여 네트워크를 형성하고 소통할 수 있는 모바일 앱입니다.

같은 고등학교라는 공통된 배경을 가진 학생들이 보다 쉽게 연결되고 서로의 관심사나 활동을 공유하며 의미있는 관계를 맺을 수 있는 플랫폼을 제공하고자 개발하였습니다.

이 프로젝트는 학교라는 제한된 물리적 공간에서 벗어나 학생들 간의 소통 범위를 확장하고자 하는 아이디어에서 출발했습니다.

같은 학교에 다니면서도 서로 모르고 지나치는 학생들이 많다는 점에 주목했고, 공통의 관심사나 활동을 통해 새로운 인맥을 형성할 수 있는 디지털 공간의 필요성을 느꼈습니다.

단순한 채팅을 넘어 학생들이 자신의 프로젝트나 활동을 소개하고, 비슷한 관심사를 가진 친구, 동료를 찾아 학교에서도 협업을 할 수 있는 환경을 만들고 싶었습니다.

### 2. 시스템 구성

모바일 앱을 구성하기 위하여 `React Native`를 사용했습니다.

웹 기반 관리자 도구, 사용자 지원을 위해 `Next.js`와 앱의 서버 구축을 위해 `Nest.js`를 사용하였습니다.

데이터베이스는 `MongoDB`를 사용하였습니다.

Pickle은 아래와 같은 화면들로 구성되어 있습니다.

| 화면명      | 역할                                            |
| ----------- | ----------------------------------------------- |
| 홈 화면     | 전체 게시글, 학년 게시판, 글 쓰기, 검색 등 제공 |
| 라이브 화면 | 사용자들의 스토리 형태 게시물 제공              |
| 채팅 화면   | 타인과의 채팅 기능 제공                         |
| 프로필 화면 | 프로필, 설정 등에 접근                          |

### 3. 프로젝트 회고

구조도를 보시면 알 수 있듯, 정말 열심히 준비했던 프로젝트입니다.

개발 뿐만 아니라 디자인과 차후 운영 방식까지 로드맵을 설계하였지만, 고등학교 3학년으로 진급하여 수능 공부에 전념해야 하는 시기가 찾아오면서 앱의 Frontend 개발까지만 완료하고, Backend 개발에서 멈춰 아쉬움이 컸던 프로젝트입니다.

차후 운영을 위한 로드맵과 제품 정의서를 작성해보면서, 개발만으로 알지 못했던 운영 및 기획의 중요성을 알 수 있었습니다.
