### 1. 개요

최근 피싱 사이트가 급증하며 인터넷 사용자들에게 노출된 피싱 위험성이 증가하였습니다.

피싱 사이트의 경우 일반인이 구별하기 힘들고, 모든 연령대에서 피해가 발생할 수 있습니다.

이러한 배경으로 KISA가 악성 코드로 인해 가짜 사이트로 이동된 경우 탐지, 안내하는 서비스인 파밍알리미를 운영하고 있으며, 이를 통해 파밍 범죄를 80%이상 감소시켰습니다.

이는 피싱, 파밍 피해의 주요 원인이 사용자가 사이트의 위험을 인지하지 못하는 데 있음을 보여준다고 볼 수 있습니다.

이러한 선행 사례를 바탕으로, 사용자가 사이트에 접근하기 전 위험 여부를 사전에 인지할 수 있도록 지원함으로써, 피싱 사이트 접속으로 인한 피해를 유의미하게 감소시키는 것을 목표로 RECON을 개발하였습니다.

### 2. 시스템 구성

본 프로젝트는 브라우저 익스텐션 형태로 제공되며 이를 위해 `React` 및 `Vite` 기반 [Jonghakseo/chrome-extension-boilerplate-react-vite](https://github.com/Jonghakseo/chrome-extension-boilerplate-react-vite)를 사용하였습니다.

익스텐션에서의 정보 수집 및 저장을 위해 `Prisma`를 사용한 `Nest.js`로 백엔드 서버를 구축하였습니다.

데이터베이스의 경우 `Supabase`를 사용하였습니다.

RECON은 아래와 같은 서비스 구조를 가지고 있습니다.

| 서비스          | 역할                                                                                                                                                                                            |
| --------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 익스텐션        | 사용자와 접촉하는 사이트들의 데이터를 서버로 전송, 받은 분석 결과 표시                                                                                                                          |
| 백엔드 서버     | 익스텐션으로부터 받은 데이터를 피싱 판별 모델에 전달, 피싱 판별 모델로부터 받은 분석 결과를 익스텐션에 전송, 분석 결과를 데이터베이스에 저장, 관리자 대시보드에 데이터베이스에 저장된 내용 제공 |
| 피싱 판별 모델  | 서버에서 전송받은 데이터를 바탕으로 분석 진행 후, 서버에 전달                                                                                                                                   |
| 관리자 대시보드 | 서버로부터 받은 데이터베이스에 저장된 분석 내용 기반으로 피싱 사이트 현황, 위험도 통계, 상세 분석 등을 제공하여 서비스 이용과 운영 지원                                                         |
| 데이터베이스    | 서버로부터 받은 분석 결과 저장, 저장 내용 서버에 전달, 분석 결과의 경우 피싱 사이트의 위험도와 판별 사유로 구성되며, 축적된 데이터는 향후 모델 고도화 및 관계 기관과의 협업에 활용              |

### 3. 프로젝트 회고

본 프로젝트는 DACON에서 개최한 ["피싱·스캠 예방을 위한 서비스 개발 경진대회"](https://dacon.io/competitions/official/236666/leaderboard)에 참여하기 위해 개발되었습니다.

대회 참가를 위해 팀을 빌딩하고 팀원들과 열심히 기획, 디자인, 개발을 진행하였지만, 아쉽게도 수상을 하지 못했습니다.

그래도 익스텐션이라는 새로운 형태의 앱을 개발해볼 수 있었던 것과 `Prisma`를 처음 사용해본 점에서 성장에 큰 도움이 되었던 프로젝트입니다.
